# 07. Single Agent 기반 AX Compass 커리큘럼 설계 에이전트

## 개요

`07.SingleAgent`는 `05.Advanced_RAG`에서 만든 검색/생성 기능을 바탕으로,
하나의 Agent가 대화형으로 정보를 수집하고 필요한 도구를 선택해 커리큘럼을 설계하는 구조입니다.

핵심은 다음과 같습니다.

- 사용자의 요구사항을 대화로 수집
- 필요하면 내부 RAG 검색 수행
- 필요하면 외부 검색(Tavily) 수행
- 수집한 문맥을 바탕으로 OpenAI 모델이 최종 커리큘럼 생성

---

## 05.Advanced_RAG와의 차이

| 항목 | 05.Advanced_RAG | 07.SingleAgent |
|------|------------------|----------------|
| 구조 | 고정 파이프라인 (`/chat`, `/generate`) | 단일 대화 루프 (`/chat`) |
| LLM | OpenAI `gpt-4o-mini` | OpenAI tool use 기반 Agent |
| 흐름 제어 | 코드가 순서를 고정 | Agent가 상황에 따라 도구 선택 |
| 외부 검색 | 없음 | Tavily 웹 검색 가능 |
| 수정/재생성 | 제한적 | 대화 기반 반복 수정 가능 |

## Agent 도구

| 도구 | 설명 |
|------|------|
| `web_search` | Tavily API를 사용한 외부 웹 검색 |
| `search_ax_compass` | 내부 VectorDB에서 AX Compass 유형별 특징 검색 |
| `search_curriculum_examples` | 내부 VectorDB에서 강의 커리큘럼 예시 검색 |
| `generate_curriculum` | 수집한 문맥을 바탕으로 최종 커리큘럼 생성 |

---

## Agent 동작 흐름

```text
사용자 메시지
    ↓
OpenAI Agent
    ├─ 추가 정보가 필요하면 질문
    ├─ 내부 AX Compass 검색
    ├─ 내부 커리큘럼 예시 검색
    ├─ 필요 시 Tavily 외부 검색
    └─ generate_curriculum 호출
            ↓
      최종 커리큘럼 생성
            ↓
   reply + curriculum 반환
```

클라이언트는 매 요청마다 전체 대화 기록을 서버에 보내고,
Agent는 그 기록을 바탕으로 이번 턴에 어떤 도구를 사용할지 판단합니다.

## 실행

### 로컬 실행

백엔드:

```bash
uvicorn main:app --app-dir 07.SingleAgent --host 0.0.0.0 --port 8000
```

프런트:

```bash
streamlit run 07.SingleAgent/07_7.Streamlit.py
```

### Docker 실행

```bash
docker-compose -f 07.SingleAgent/07_9.docker-compose.yml up --build
```

- 백엔드: `http://localhost:8000`
- 프런트: `http://localhost:8501`
